This means the password reset didn't work, or pgAdmin is connected with a different method. Let's force-reset it:

**Step 1 — Find your `pg_hba.conf` file.** In pgAdmin Query Tool run:
```sql
SHOW hba_file;  C:/Program Files/PostgreSQL/14/data/pg_hba.conf
```
It will return a path like `C:/Program Files/PostgreSQL/14/data/pg_hba.conf`

**Step 2 — Open that file in Notepad (as Administrator) and find lines like:**
```
host    all    all    127.0.0.1/32    scram-sha-256
host    all    all    ::1/128         scram-sha-256
```
Change `scram-sha-256` to `trust` on both lines:
```
host    all    all    127.0.0.1/32    trust
host    all    all    ::1/128         trust
```

**Step 3 — Restart PostgreSQL service.** Open Windows Services (`Win + R` → `services.msc`), find **postgresql-x64-14**, right-click → **Restart**.

**Step 4 — Now connect without a password:**
```bash
psql -U postgres -h localhost
```
You should get in without any password prompt.

**Step 5 — Reset the password:**
```sql
ALTER USER postgres WITH PASSWORD 'postgres123';
\q
```

**Step 6 — Go back to `pg_hba.conf`, revert `trust` back to `scram-sha-256`, and restart the service again.**

**Step 7 — Update `database.py`:**
```python
"postgresql://postgres:postgres123@localhost:5432/shoplite"
```

Now `psql -U postgres -h localhost` should work with `postgres123`.

In your file, modify **these lines**:

```diff

- local   all             all                                     scram-sha-256
+ local   all             all                                     trust
```

Optionally (only if you're connecting via localhost TCP, not socket):

```diff
- host    all             all             127.0.0.1/32            scram-sha-256
```
